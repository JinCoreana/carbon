# Help
Typically triggered by the user hovering on an icon, displaying a single, short sentence. 
Useful for explaining the difference between commands represented with icons, for example, batch table actions. 
Bear in mind that tooltips may not work well (or at all) on mobile devices, so alternatives may be needed.

## Quick Start
Import `Help` into the project.

```jsx
import Help from "carbon-react/lib/components/help";

const MyComponent = () => (
  <Help>
    Some helpful text goes here
  </Help>
);

```

### Help with Tooltip Position
The tooltip can be position `right`, `left`, `top` or `bottom`.
Hover over the help component to see the different positions.

<Preview>
  <Story name="with Tooltip Position">
    {() => {
      return ['right', 'left', 'top', 'bottom'].map(position =>
        <div style={ { margin: '64px' } }>
          <Help tooltipPosition={`${position}`} isFocused={true}>
            This tooltip is positioned {`${position}`}
          </Help>
        </div>
      )
    }}
  </Story>
</Preview>

### Help with Tooltip Align
The tooltip alignment can be `center`, `bottom`, `left`, `right` or `top`.
Hover over the help component to see the different alignments.

<Preview>
  <Story name="with Tooltip Align">
    {() => {
      return [
        'center',
        'left',
        'right'
        ].map(align =>
        <div style={ { margin: '64px' } }>
          <Help 
          tooltipPosition='top' 
          tooltipAlign={`${align}`}
          isFocused={true}>
            This tooltip is aligned {`${align}`} 
          </Help>
        </div>        
      )
    }}
  </Story>
</Preview>

`tooltipAlign` top and `tooltipAlign` bottom only accepts `tooltipPosition` left or right. In this example, tooltipPosition is set to `right`.
Hover over the help component to see the different alignments.
